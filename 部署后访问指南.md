## 📌 快速部署（3 步完成）

### 第 1 步：登录服务器

```bash
ssh root@你的服务器 IP
```

### 第 2 步：克隆并配置

```bash
# 克隆代码
git clone <您的仓库地址>
cd xiang

# 配置环境变量
cp .env.example .env
nano .env
```

**必须修改的配置：**
```bash
# 1. JWT_SECRET - 执行以下命令生成随机字符串
openssl rand -base64 32
# 将输出值复制到 .env 中的 JWT_SECRET

# 2. DB_PASSWORD - 建议修改为强密码（可选）
DB_PASSWORD=your_secure_password
```

### 第 3 步：一键启动

```bash
# 启动服务
docker-compose up -d --build

# 查看状态
docker-compose ps

# 查看日志（可选）
docker-compose logs -f app
```

## 🔍 故障排查

### 服务启动失败

```bash
# 查看详细错误
docker-compose logs app

# 检查环境变量
cat .env
```

### 数据库连接失败

```bash
# 检查数据库容器
docker-compose ps postgres

# 查看数据库日志
docker-compose logs postgres

# 重启数据库
docker-compose restart postgres
```

### 端口被占用

```bash
# 检查端口占用
netstat -tunlp | grep 5700

# 修改 docker-compose.yml 端口映射
ports:
  - "5701:5700"  # 改为其他端口
```

## 🔐 安全建议

1. **修改 JWT_SECRET** - 使用至少 32 字符的随机字符串
2. **修改管理员密码** - 首次登录后立即修改
3. **配置防火墙** - 仅开放 5700 端口
4. **使用 HTTPS** - 生产环境建议配置 SSL 证书
5. **定期备份数据库** - 详见 `DEPLOY_SERVER.md`
