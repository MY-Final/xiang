# 恋爱博客架构指南（定稿）

本指南用于统一恋爱博客项目的技术选型、目录结构、模块职责和部署方式。

## 1. 技术栈（最终版）

- 前端：Vue 3 + TypeScript + Vite + Element Plus + Pinia + Vue Router + Axios
- 后端：Go + Gin + GORM + JWT
- 数据库：PostgreSQL
- API 文档：Swagger / OpenAPI（swaggo）
- 文件存储：本地 `uploads/`（后续可切 OSS/COS/S3）
- 部署：Nginx + 前端静态站点 + Go API + PostgreSQL

说明：
- UI 组件库统一使用 **Element Plus**。
- 样式方案可以保留基础 CSS/SCSS；如需工具类，可按需引入 Tailwind，但不是必选。

---

## 2. 架构原则

- 前后端分离：前端仅通过 HTTP API 访问后端。
- 同仓库管理：前后端放在同一个仓库根目录，便于协作和部署。
- 分层设计：后端采用 Controller / Service / Repository 分层。
- 职责清晰：前端负责展示与交互，后端负责鉴权与业务。

## 8. 关键流程

### 8.1 管理员登录

1. 管理员输入账号密码。
2. 前端请求 `POST /api/v1/auth/login`。
3. 后端校验通过后返回 JWT。
4. 前端保存 token（建议 `localStorage`）。
5. 后续管理接口带 `Authorization: Bearer <token>`。

### 8.2 文章发布

1. 管理员在 `/admin` 进入文章编辑页。
2. 富文本/Markdown 编辑内容。
3. 图片调用上传接口。
4. 提交 `POST /api/v1/admin/posts`。
5. 后端写入数据库，前台通过公开接口读取展示。

### 8.3 站点展示

1. 用户访问站点首页。
2. 前端调用 `GET /api/v1/posts`。
3. 后端查询并返回文章列表。
4. 前端渲染列表与详情。

## 10. 开发环境约定

- 前端：`http://localhost:6100`
- 后端：`http://localhost:8979`
- 数据库：默认 `localhost:5432`，也支持通过环境变量连接远端 PostgreSQL（云服务器/托管数据库）

Vite 代理示例：

```ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8979',
      changeOrigin: true,
    },
  },
}
```

## 11. 生产部署建议

```text
Nginx
├── /           -> frontend/dist
├── /api/       -> backend:8979
└── /uploads/   -> backend/uploads

Go API 服务
└── 监听 8979

PostgreSQL
└── 数据存储
```

建议：
- 使用同域名统一入口，减少跨域复杂度。
- 通过 Nginx 反向代理 API 与静态资源。
- 数据库可与 API 分离部署：可使用独立服务器或托管 PostgreSQL，仅需放通网络并配置连接环境变量。

## 13. 环境变量配置建议

推荐在仓库根目录维护 `.env.example` 作为模板，实际运行时复制为 `.env` 并填写真实值。

示例（支持本地或远端 PostgreSQL）：

```env
# app
APP_ENV=development
SERVER_HOST=0.0.0.0
SERVER_PORT=8979

# postgres
DB_HOST=127.0.0.1
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=change_me
DB_NAME=love_blog
DB_SSLMODE=disable
DB_TIMEZONE=Asia/Shanghai

# jwt
JWT_SECRET=change_me_to_a_long_random_string
JWT_EXPIRE_HOURS=72
```

说明：
- 本地开发通常使用 `DB_HOST=127.0.0.1`。
- 若使用服务器/托管 PostgreSQL，将 `DB_HOST` 改为远端地址（域名或 IP），并按服务商要求调整 `DB_PORT`、`DB_SSLMODE`、白名单与证书配置。
