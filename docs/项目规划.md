## 一、项目概述

「湘」遇恋爱博客系统 —— 一个浪漫、温馨、有情感温度的全栈博客平台。

### 技术栈

| 层级 | 技术选型 |
|------|----------|
| **前端** | Vue 3 + TypeScript + Vite + Pinia + Vue Router + Element Plus |
| **后端** | Go + Gin + GORM + Redis + PostgreSQL |
| **认证** | JWT (JSON Web Token) |
| **API 规范** | RESTful + Swagger 文档 |

### 端口配置

| 服务 | 端口 | API 前缀 |
|------|------|----------|
| 前端 | 6100 | - |
| 后端 | 8979 | `/api/v1` |

## 三、Git 工作流

### 分支策略

```
main                    # 生产分支（受保护，只能从 develop 合并）
  └── develop           # 开发主分支
      ├── feature/      # 新功能分支
      ├── fix/          # Bug 修复分支
      └── style/        # 代码格式化分支
```

### 分支命名规范

| 分支类型 | 格式 | 示例 |
|----------|------|------|
| 新功能 | `feature/<功能英文>` | `feature/user-auth` |
| Bug 修复 | `fix/<问题描述>` | `fix/token-expiration` |
| 重构 | `refactor/<模块>` | `refactor/auth-controller` |
| 文档 | `docs/<内容>` | `docs/api-auth` |
| 测试 | `test/<内容>` | `test/auth-api` |
| 格式化 | `style/formatting` | `style/formatting` |

### 提交规范（Conventional Commits）

```
<type>(<scope>): <subject>
```

**Type 取值：**
- `feat`: 新功能
- `fix`: Bug 修复
- `refactor`: 重构
- `docs`: 文档更新
- `test`: 测试相关
- `style`: 格式化（不影响代码逻辑）
- `chore`: 构建/工具配置

**Scope 取值：**
- `auth`: 认证模块
- `user`: 用户模块
- `post`: 文章模块
- `frontend`: 前端通用
- `backend`: 后端通用
- `deps`: 依赖管理

**提交示例：**
```bash
feat(auth): add user login API
fix(frontend): resolve token refresh issue
docs(api): update swagger auth endpoints
style(frontend): format login page with prettier
chore(deps): update vue-router to v4.3.0
```

### 合并流程

```
1. 从 develop 创建 feature 分支
   git checkout develop
   git checkout -b feature/user-auth

2. 开发完成后合并到 develop
   git checkout develop
   git merge --no-ff feature/user-auth

3. 删除已合并的 feature 分支
   git branch -d feature/user-auth
```

## 四、API 接口规范

### 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 错误码定义

| Code | 说明 |
|------|------|
| 0 | 成功 |
| 4001 | 请求参数错误 |
| 4004 | 资源不存在 |
| 4009 | 资源冲突（如用户名已存在） |
| 4010 | 未授权/Token 无效 |
| 4030 | 禁止访问 |
| 5000 | 服务器内部错误 |

### 认证机制

所有需要认证的接口需在请求头携带：
```
Authorization: Bearer <token>
```

Token 过期时间：72 小时（可配置）

## 六、开发规范

### Go 代码规范

```go
// 命名规范
// ✅ 导出标识符：PascalCase
type PostService struct {}

// ✅ 非导出标识符：camelCase
var postRepo repository.PostRepository

// 错误处理
if err != nil {
    return fmt.Errorf("create post: %w", err)
}

// 上下文传递
func (s *postService) Create(ctx context.Context, dto *dto.CreatePostReq) (*Post, error)
```

### Vue 代码规范

```typescript
// 使用 Composition API + <script setup>
<script setup lang="ts">
import { ref, computed } from 'vue'
import { useUserStore } from '@/stores/user'

const userStore = useUserStore()
const isLoading = ref(false)

const handleLogin = async () => {
  isLoading.value = true
  try {
    await userStore.login(credentials)
  } finally {
    isLoading.value = false
  }
}
</script>
```
