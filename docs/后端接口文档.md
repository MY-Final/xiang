* 前后端分离
* 后端：**Go + Gin**
* 认证方式：**JWT**
* 接口前缀：`/api/v1`
* 分为：

    * 公开接口
    * 认证接口
    * 管理后台接口

我会按“能直接拿去开发”的方式写。

---

# 一、接口分组总览

## 1）公开接口

给前台页面使用，不需要登录：

* 获取文章列表
* 获取文章详情
* 获取标签列表
* 获取分类列表
* 获取时间线
* 获取站点设置

## 2）认证接口

给后台登录使用：

* 管理员登录
* 获取当前管理员信息
* 退出登录

## 3）后台管理接口

需要管理员登录：

* 文章管理
* 标签管理
* 分类管理
* 图片上传
* 媒体库管理
* 站点设置

### 接口

```http

```

### 请求体

```json
{
  "username": "admin",
  "password": "123456"
}
```

### 响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "jwt-token",
    "expires_in": 604800,
    "user": {
      "id": 1,
      "username": "admin",
      "nickname": "myfinal",
      "avatar": ""
    }
  }
}
```

# 四、前台公开接口

## 1. 获取文章列表

### 接口

```http
GET /api/v1/posts
```

### 查询参数

| 参数          | 类型     | 说明            |
| ----------- | ------ | ------------- |
| page        | int    | 页码            |
| page_size   | int    | 每页数量          |
| keyword     | string | 关键词搜索         |
| tag_id      | int    | 标签 ID         |
| category_id | int    | 分类 ID         |
| sort        | string | 排序，如 `latest` |
| is_top      | bool   | 是否优先置顶        |

### 示例

```http
GET /api/v1/posts?page=1&page_size=10
```

* `GET /api/v1/posts`
* `GET /api/v1/posts/slug/:slug`
* `GET /api/v1/tags`
* `GET /api/v1/categories`

## 后台

* `GET /api/v1/admin/posts`
* `GET /api/v1/admin/posts/:id`
* `POST /api/v1/admin/posts`
* `PUT /api/v1/admin/posts/:id`
* `DELETE /api/v1/admin/posts/:id`
* `POST /api/v1/admin/tags`
* `GET /api/v1/admin/tags`
* `POST /api/v1/admin/categories`
* `GET /api/v1/admin/categories`
* `POST /api/v1/admin/upload/image`

# 十二、推荐的 Swagger 分组写法

你后面做 Swagger 文档时，可以按这些 Tag 分：

* `Auth`
* `Post`
* `AdminPost`
* `Tag`
* `Category`
* `Upload`
* `Media`
* `Site`

这样文档会很清楚。
